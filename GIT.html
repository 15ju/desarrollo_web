Versionado

Es muy recomendable trabajar con herramientas de versionado, ya que, por ejemplo, cuando modifico un html que estoy usando en visual code, no me queda la version previa a las modificaciones que haya introducido.

Las herramientas de versionado(como GIT) me dejan ir modificando el dise√±o que estoy realizando pero manteniendo la posibilidad de volver a ver o usar la version anterior a las modificaciones que le realicemos.

Cada vez que hacemos una modificacion en un archivo y vemos que lo vamos a dejar, vamos a escribir en el repositorio local para que guarde como esta el archivo, esto me permite ver como estaba el archivo en el momento que quiera.

  1. Se va a inicializar con "git init", que va a iniciar y crear un directorio nuevo donde va a ir guardando los cambios.
  2. Ponemos "git config" para poner datos de configuracion, si no decimos quienes somos despues el git no nos va a dejar modificar los archivos porque no va a autenticarnos correctamente
    1. se pone git config user.name "nombre"
    2. git config user.email "email"

Esto se hace para que cuando se suba el documento al repositorio remoto, donde varias personas van a interactuar con el mismo, se pueda ver quienes estan trabajando o modificando el archivo que esta ahi subido

Lenguaje de programacion

Es un lenguaje formal que proporciona la capacidad de escribir una secuencia de ordenes en forma de algoritmos, que van a producir una salida o ejecutar una tarea.

Tipos de lenguaje de programacion

Hay muchos lenguajes de programacion, diferentes entre si, pero comparten la logica necesaria para desarrollar la solucion a un problema, usan el mismo algoritmo generico(pseudocodigo)

Programar es el proceso de crear un software viable y el mantenimiento del codigo fuente. Se aplican los siguientes pasos:

    Caracteristicas de un lenguaje de programacion

  • Desarrollo logico para resolver un problema
  • Escritura de la logica empleando un lenguaje de programacion especifico (codificacion)
  • Compilacion del programa hasta convertirlo en lenguaje de maquina
  • Prueba y depuracion
  • Desarrollo de la documentacion

El conjunto de simbolos que conforman el lenguaje se llama alfabeto

Debug es depuracion: es encontrar el error que tiene el codigo fuente, pudiendo ser sintaxis (error de escritura) y problemas que tiene cuando se ejecuta. Va a mostrar como se hace hablar al codigo, a medida que se va ejecutando muestra el resultado que da.

Pesima practica ir a tocar un programa que esta corriendo sin problemas. Si no tiene bugs no tiene sentido tocar el codigo. Un codigo puede ser modificado en el caso de que necesite mejoras.

Hay que estar atento de escribir los lenguajes de manera correcta, porque es estricto, si esta mal escrito va a arrojar un error directamente.

En este modulo trabajamos con javascript, es el lenguaje que se utiliza para frontend y backend, es para que el usuario pueda darle ordenes al codigo. Le dice a la computadora que queremos que haga. Entro, el navegador interpreta y ejecuta lo que js le haya establecido.

Interpretes y compiladores

Un compilador es un programa que agarra el codigo fuente y lo baja a codigo de maquina (assembly).

Java es un lenguaje que compila, lo que se hace es transformar el codigo fuente, que es el java, en un codigo que la maquina entiende.

Interprete: Traduce el programa desde su descripcion a medida que se va ejecutando.

Si uso un compilador y algo esta mal, va a decir que hay un error y no se puede compilar, ademas va a decir que hay un error en una linea. En cambio, si ejecuto el codigo a traves de un interprete, este va a ejecutar el mismo y en el caso de que haya un error va a crashear en ese punto, pero habiendo ejecutado lo anterior.

En los lenguajes que son interpretados hay que chequearlos mucho antes de probarlos

El interprete ejecuta la secuencia que uno le da y en el caso de que haya un fallo va a crashear en ese momento, pero habiendo tratado de ejecutarlo todo.

Lenguajes de programacion

Clasificacion de los lenguajes de la programacion

    Hay varias clasificaciones, la que le presta atencion es

  • De alto nivel
  • De bajo nivel

    • Se refieren a su nivel de abstraccion, en cuanto a lo especifico o general que es respecto a la arquitectura de la computadora, dependiendo del lenguaje que usemos, vamos a estar mucho mas cerca de hablarle al ordenador de forma directa. Cuando estamos mas lejos se habla de lenguajes de alto nivel, cuando estamos mas cerca, lenguaje de bajo nivel

Clasificacion por paradigma

    Hay dos muy marcados

  • Paradigma procedural
  • Orientado a objetos
  • Programacion dirigida por eventos

Procedural seria la descripcion de la receta hasta llegar al resultado final.

Orientado a objetos: Esta todo mucho mas estructurado

Segun el lenguaje con el que trabajemos va a seguir uno u otro paradigma.

Hay que tener en cuenta el tipo de arquitectura que tiene el equipo en el que vamos a usar el programa que desarrollamos, para asegurarse que sea compatible con lo que desarrollamos.

La programacion dirigida por eventos esta tomando mucho auge, cobro mucha importancia por el cloud computing, hay que prestarle atencion, saber que esta ahi.
